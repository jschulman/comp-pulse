# Contributing

Comp Pulse is a single-author project maintained by Jay Schulman. It's open-source so that anyone can verify the methodology and rebuild the index from scratch if they want to — not because it's accepting external code contributions.

## What's welcome

- **Issues** that flag genuine methodology problems: a missing regex pattern for an unusual comp-range format, a calculation bug in the deltas, a flaw in the disclosure-rate denominator. Be specific.
- **PRs** for typo fixes, accessibility improvements to the dashboard, or documentation clarifications.
- **Discussion** about whether the canary should track p50 vs p25 vs midpoint of range.

## What's not welcome

- PRs that change the index methodology without prior discussion in an issue. The methodology is intentionally simple.
- PRs that add new external dependencies. The dashboard is intentionally static.
- Feature requests for specific company comp data. The canary is sub-sector aggregated and direction-only by design.

## How the data is generated

The data file `data/latest.json` and the per-day snapshots in `data/snapshots/` are generated by a private pipeline that polls public job-board APIs (Greenhouse, Ashby) and aggregates disclosed compensation ranges via the methodology in [METHODOLOGY.md](METHODOLOGY.md). The pipeline's source is not in this repo. If you want to reproduce the data, follow the methodology — don't reverse-engineer the JSON.
